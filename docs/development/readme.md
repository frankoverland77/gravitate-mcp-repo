# 🛠️ Development Documentation

This directory contains internal development rules, guidelines, and templates for maintaining and extending the Excalibrr MCP Server.

## 📋 Files Overview

### **CODE_GENERATION_RULES.md**

- **Purpose**: Operational rules for the MCP server and generated projects
- **Audience**: Developers working on the MCP server
- **Content**: Server management rules, port handling, project generation patterns

### **example-coding-conventions.md**

- **Purpose**: Coding standards and patterns for component examples
- **Audience**: Developers adding new component examples
- **Content**: File structure conventions, TypeScript patterns, documentation standards

### **example-quality-checklist.md**

- **Purpose**: Quality assurance checklist for component examples
- **Audience**: Developers and reviewers
- **Content**: Validation criteria, testing requirements, production standards

### **standardized-example-template.md**

- **Purpose**: Template for creating new component examples
- **Audience**: Developers adding new components
- **Content**: Boilerplate code, structure templates, documentation patterns

## 🎯 When to Use These Files

- **Before adding new components**: Review coding conventions and use the template
- **During code review**: Use the quality checklist to validate examples
- **When debugging server issues**: Refer to code generation rules
- **For onboarding new developers**: Start with these guidelines

## 📁 Organization

These files are separated from user documentation because they are:

- **Internal development guidelines** (not user-facing)
- **Technical implementation details** (not setup instructions)
- **Code quality standards** (not usage examples)

User-facing documentation remains in the parent `/docs/` directory.
